-Database
Database adalah tempat menyimpan collection(table)
Semua collection harus disimpan di database
Biasanya database digunakan untuk memisahkan data secara logica peraplikasi artinya biasanya satu aplikasi akan memiliki satu database,
Jarang sekali kita akan menggunakan satu database untuk beberapa aplikasi.

-Membuat Database
Kita tidak perlu secara eksplisit membuat database,
MonggoDB akan secara otomatis membuat database sesuai dengan nama database yang kita pilih,
Untuk memilih nama database kita bisa menggunakan perintah "USE"

-collection
Collection adalah tempat untuk menyimpan document
Maximum perDocument yang bisa disimpan adalah 16MB
Maximum level nested document yang bisa disimpan adalah 100 Level
